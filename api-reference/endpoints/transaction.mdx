# Transaction API

The `/transaction` endpoint allows merchants to process **Card** or **ACH** transactions using AUXVAULT's public API. The API supports one-time sale or authorization-based payments and can optionally tokenize payment methods for future use.

## Card Transaction

### API Endpoint
`https://dev-api.auxvault.net/api/v1/public/transaction`

### Request Parameters

The request must be a POST request with the following parameters:

| Parameter              | Data Type | Required | Description                                |
|------------------------|-----------|----------|--------------------------------------------|  
| Amount                | Decimal   | Yes      | The amount of the transaction              |
| BillingAddress        | String    | No       | The billing addresses                      |
| BillingCity           | String    | No       | The billing city                           |
| BillingCountry        | Integer   | Yes      | The billing country                        |
| BillingCountryCode    | String    | Yes      | The billing country code                   |
| BillingCustomerName   | String    | Yes      | The name of the billing customer           |
| BillingEmail          | String    | Yes      | The billing email                          |
| BillingPhoneNumber    | String    | Yes      | The billing phone number                   |
| BillingPostalCode     | String    | Yes      | The billing postal code                    |
| BillingState          | Integer   | No       | The billing state                          |
| ShippingSameAsBilling | Boolean   | No       | For differentiating address                |
| ShippingCustomerName  | String    | No       | The name of the shipping customer          |
| ShippingEmail         | String    | No       | Email address of shipping customer         |
| ShippingPhoneNumber   | String    | No       | Phone number of shipping customer          |
| ShippingAddress       | String    | No       | The shipping Address                       |
| ShippingCountry       | String    | No       | Shipping country                           |
| ShippingCity          | String    | No       | Shipping city                              |
| ShippingState         | String    | No       | Shipping state                             |
| ShippingPostalCode    | String    | No       | Shipping postal code                       |
| CardNumber            | String    | Yes      | The card number                            |
| Cvv                   | String    | Yes      | The card verification value                |
| ExpiryDate            | String    | Yes      | The card expiry date                       |
| SuggestedMode         | String    | Yes      | The suggested payment mode || Card         |
| ConvenienceFeeActive  | Boolean   | Yes      | Whether or not to activate the convenience fee || true / false |
| TransactionType       | String    | No       | The transaction type is 1 means it is Sale and transaction type is 2 means it is Auth transaction |
| PaymentTokenization   | Boolean   | No       | For tokenization of the card/ach details for future payments |
| IpAddress             | String    | Yes      | IPv4 or IPv6 value of the end user         |

### Example Request
```json
{ 
  "Amount": 1000, 
  "BillingAddress": "", 
  "BillingCity": "", 
  "BillingCountry": "US",
  "BillingCountryCode": "+1",
  "BillingCustomerName": "xxxxxxxxxxxxx", 
  "BillingEmail": "xxxxxxxxxxxx@gmail.com", 
  "BillingPhoneNumber": "897>void 8976732233", 
  "BillingPostalCode": "54321", 
  "BillingState": "", 
  "CardNumber": "4242424242424242", 
  "Cvv": "123", 
  "ExpiryDate": "12/24",
  "SuggestedMode": "Card", 
  "ConvenienceFeeActive": true,
  "TransactionType": "1",
  "PaymentTokenization": true,
  "IpAddress": "192.60.32.233"
}
```

### Response
The response will contain the status, message, and data of the transaction.

| Parameter     | Data Type | Description                              |
|---------------|-----------|------------------------------------------|
| status        | String    | The status of the transaction            |
| status_code   | Integer   | For identify the response                |
| message       | String    | A message describing the transaction status |
| data          | Object    | An object containing details of the transaction |

**Example Response**:
```json
{
    "status": "success",
    "status_code": 100,
    "message": "Transaction Processed Successfully",
    "data": {
        "TransactionId": "d0snjfv0i479rlkc66v0",
        "BaseAmount": 10,
        "Amount": 10,
        "CardNumber": "1111",
        "BillingEmail": "federico@gmail.com",
        "BillingCustomerName": "Federico",
        "BillingAddress": "",
        "BillingCity": "",
        "BillingState": null,
        "BillingPostalCode": "",
        "BillingCountry": 1,
        "BillingCountryCode": "+1",
        "BillingPhoneNumber": "876543200",
        "TransactionType": 1,
        "Status": "1"
    }
}
```

### Error Codes
If an error occurs, the response will contain an error code and a message.

| Error Code | Description               |
|------------|---------------------------|
| 400        | Bad Request               |
| 401        | Unauthorized              |
| 403        | Forbidden                 |
| 404        | Not Found                 |
| 500        | Internal Server Error     |

### Security
To access this API endpoint, you will need to provide an API key in the header:

```javascript
var request = { 
  method: "post", 
  url: "https://dev-api.auxvault.net/api/v1/public/transaction", 
  headers: { 
    "Content-Type": "application/json", 
    "Authorization": "APIKEY"
  }, 
  data: data
}
```

### Transaction Status Codes

| Status              | Value |
|---------------------|-------|
| Pending             | 0     |
| Approved            | 1     |
| Failed              | 2     |
| Captured            | 3     |
| Settled             | 4     |
| Partially Returned  | 5     |
| Initiated           | 6     |
| Payment Captured    | 7     |
| Voided              | 8     |
| Declined            | 9     |
| Fully Returned      | 10    |
| Paid Out            | 11    |
| Reversed            | 12    |
| Chargeback          | 13    |
