---
title: 'Void Transaction API'
description: 'Void authorized transactions before they are settled'
---

This endpoint allows you to void (cancel) a previously authorized transaction before it settles.

**Note**: The production endpoint for this operation is `https://api.auxvault.net/api/v1/public/transaction`.

<Endpoint method="POST" url="https://dev-api.auxvault.net/api/v1/public/void">

<ParamField type="header" name="Authorization" required>
  Your API key for authentication.
</ParamField>

<ParamField body="TransactionId" type="string" required>
  The unique identifier of the transaction to be voided.
</ParamField>

<ParamField body="TransactionFor" type="string" required>
  The type of transaction. Must be set to `Void`.
</ParamField>

<ParamField body="Reason" type="string">
  An optional reason for voiding the transaction.
</ParamField>

</Endpoint>

## Example Request

```json
{ 
  "TransactionId": "chcfbh46lyu7653qe",
  "TransactionFor": "Void",
  "Reason": "Customer request"
}
```

## Example Response

<Response>

```json
{
    "status": "success",
    "status_code": 200,
    "message": "Transaction Void Initiated Successfully",
    "data": {
        "TransactionId": "chcfbh46lr8uj3go8t30",
        "type": "Void"
    }
}
```

</Response>
