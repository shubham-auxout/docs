# Transaction Retrieval API

The Transaction Retrieval API allows you to fetch information about previously processed transactions. You can retrieve individual transactions by ID, list transactions with various filters, and perform advanced searches.

## Get Transaction

Retrieve details about a specific transaction by its ID.

### API Endpoint

```
GET /api/v1/public/transaction/{transactionId}
```

### Path Parameters

<ParamField path="transactionId" type="string" required>

</ParamField>

## List Transactions

Retrieve a paginated list of transactions with basic filtering options.

### API Endpoint

```
GET /api/v1/public/transactions
```

### Query Parameters

<ParamField query="limit" type="integer" default="25">
  Number of results per page (max 100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip
</ParamField>

<ParamField query="startDate" type="string">
  Filter by transaction date (ISO 8601 format)
</ParamField>

<ParamField query="endDate" type="string">
  Filter by transaction date (ISO 8601 format)
</ParamField>

<ParamField query="status" type="string">
  Filter by transaction status (e.g., `approved`, `declined`, `pending`, `voided`, `refunded`)
</ParamField>

<ParamField query="type" type="string">
  Filter by transaction type (e.g., `CARD`, `ACH`)
</ParamField>

<ParamField query="customerId" type="string">
  Filter by AuxVault customer ID
</ParamField>

<ParamField query="orderId" type="string">
  Filter by your system's order ID
</ParamField>

## Search Transactions

Perform advanced searches on transactions with complex filtering.

### API Endpoint

```
POST /api/v1/public/transactions/search
```

### Request Parameters

<ParamField body="Filters" type="object" required>
  Object containing filter criteria
</ParamField>

<ParamField body="Filters.DateRange" type="object">
  Date range filter
</ParamField>

<ParamField body="Filters.DateRange.StartDate" type="string">
  Start date in ISO 8601 format
</ParamField>

<ParamField body="Filters.DateRange.EndDate" type="string">
  End date in ISO 8601 format
</ParamField>

<ParamField body="Filters.Statuses" type="array">
  Array of transaction statuses to include
</ParamField>

<ParamField body="Filters.Types" type="array">
  Array of transaction types to include
</ParamField>

<ParamField body="Filters.AmountRange" type="object">
  Amount range filter
</ParamField>

<ParamField body="Filters.AmountRange.Min" type="number">
  Minimum transaction amount
</ParamField>

<ParamField body="Filters.AmountRange.Max" type="number">
  Maximum transaction amount
</ParamField>

<ParamField body="Filters.CustomerIds" type="array">
  Array of customer IDs to include
</ParamField>

<ParamField body="Filters.OrderIds" type="array">
  Array of order IDs to include
</ParamField>

<ParamField body="Filters.CardBrands" type="array">
  Array of card brands to include (e.g., `Visa`, `Mastercard`, `Amex`)
</ParamField>

<ParamField body="Filters.Last4" type="string">
  Last 4 digits of card or bank account
</ParamField>

<ParamField body="Filters.Metadata" type="object">
  Metadata key-value pairs to match
</ParamField>

<ParamField body="Sort" type="object">
  Sorting options
</ParamField>

<ParamField body="Sort.Field" type="string" default="createdAt">
  Field to sort by
</ParamField>

<ParamField body="Sort.Direction" type="string" default="desc">
  Sort direction (`asc` or `desc`)
</ParamField>

<ParamField body="Pagination" type="object">
  Pagination options
</ParamField>

<ParamField body="Pagination.Limit" type="integer" default="25">
  Number of results per page (max 100)
</ParamField>

<ParamField body="Pagination.Offset" type="integer" default="0">
  Number of results to skip
</ParamField>

### Example Request

```json
{
  "Filters": {
    "DateRange": {
      "StartDate": "2025-06-01T00:00:00Z",
      "EndDate": "2025-07-03T23:59:59Z"
    },
    "Statuses": ["approved", "declined"],
    "Types": ["CARD"],
    "AmountRange": {
      "Min": 50.00,
      "Max": 500.00
    },
    "CardBrands": ["Visa", "Mastercard"],
    "Metadata": {
      "promotion_code": "SUMMER2025"
    }
  },
  "Sort": {
    "Field": "amount",
    "Direction": "desc"
  },
  "Pagination": {
    "Limit": 50,
    "Offset": 0
  }
}
```

### Success Response

The response format is the same as the List Transactions endpoint.

## Transaction Summary

Get summary statistics for transactions within a specified date range.

### API Endpoint

```
GET /api/v1/public/transactions/summary
```

### Query Parameters

<ParamField query="startDate" type="string" required>
  Start date in ISO 8601 format
</ParamField>

<ParamField query="endDate" type="string" required>
  End date in ISO 8601 format
</ParamField>

<ParamField query="groupBy" type="string">
  Group results by: `day`, `week`, `month`, `status`, `type`, or `cardBrand`
</ParamField>

### Success Response

```json
{
  "status": "success",
  "data": {
    "summary": {
      "totalTransactions": 157,
      "totalAmount": 12450.75,
      "averageAmount": 79.30,
      "currency": "USD",
      "statusBreakdown": {
        "approved": 142,
        "declined": 10,
        "pending": 5
      },
      "typeBreakdown": {
        "CARD": 135,
        "ACH": 22
      }
    },
    "timeRange": {
      "startDate": "2025-06-01T00:00:00Z",
      "endDate": "2025-07-03T23:59:59Z"
    }
  }
}
```

## Error Codes

| Code | Description |
|------|-------------|
| `transaction_not_found` | The specified transaction ID does not exist |
| `invalid_date_range` | The provided date range is invalid |
| `invalid_filter` | One or more filters are invalid |
| `invalid_sort_field` | The requested sort field is not supported |
| `invalid_data` | The request contains invalid data |

## Security

All requests to the Transaction Retrieval API must include your API key in the Authorization header:

```bash
Authorization: YOUR_API_KEY
```

## Data Retention

<Note>
  Transaction data is retained for the following periods:
  
  - Complete transaction records: 2 years
  - Transaction summaries: 7 years
  - After the retention period, some sensitive data may be redacted while maintaining the transaction record
</Note>
