<Endpoint method="POST" url="https://dev-api.auxvault.net/api/v1/public/transaction-list">

<ParamField type="header" name="Authorization" required>
  Your API key for authentication.
</ParamField>

<ParamField body="sort" type="string">
  Sort order for the results. Can be `asc` or `desc`.
</ParamField>

<ParamField body="page" type="integer">
  The page number for pagination.
</ParamField>

<ParamField body="perPage" type="integer">
  The number of items to return per page.
</ParamField>

</Endpoint>

## Example Request

```json
{
  "sort": "desc",
  "page": 1,
  "perPage": 10
}
```

# Transaction Details

This endpoint retrieves detailed information for a specific transaction, identified by its unique transaction ID.

<Endpoint method="GET" url="https://dev-api.auxvault.net/api/v1/public/transaction/:id">

<ParamField type="header" name="Authorization" required>
  Your API key for authentication.
</ParamField>

<ParamField type="path" name="id" required>
  The unique identifier for the transaction.
</ParamField>

</Endpoint>

# Common Error Responses

Below are common HTTP error responses you may encounter when using the Transactions API:

| Status Code | Message                                          | Description                                                                 |
|-------------|--------------------------------------------------|-----------------------------------------------------------------------------|
| 404         | "Resource not found: GET /api/v1/public/transaction-lists" | The requested endpoint or resource does not exist. Double-check the URL or endpoint path. |
| 401         | "Authorization header missing"                   | The request is missing an Authorization header. Ensure you provide a valid token. |
| 401         | "You are unauthorized to perform this action."   | The credentials provided are not authorized to access this resource. Check your token or permissions. |

**Example Error Response (JSON)**:
```json
{
  "status_code": 401,
  "message": "Authorization header missing"
}
```

## Data Retention

> Transaction data is retained for the following periods:
> 
> - Complete transaction records: 2 years
> - Transaction summaries: 7 years
> - After the retention period, some sensitive data may be redacted while maintaining the transaction record
