# Transaction List

This endpoint returns a list of transactions filtered by the provided criteria.

## API Endpoint
`https://dev-api.auxvault.net/api/v1/public/transaction-list`

## Request Parameters
The request must be a POST request with the following parameters:

| Parameter | Data Type         | Required | Description          |
|-----------|-------------------|----------|----------------------|
| sort      | String (asc / desc) | No       | for sorting the order |
| page      | Integer           | No       | For page number      |
| perPage   | Integer           | No       | For data count       |

## Example Request
```json
{
  "req.query": "sort=\"\"&page=\"\"&perPage=\"\""
}
```

## Response
The response will contain the status, message, and data of the transaction.

| Parameter | Data Type | Description                              |
|-----------|-----------|------------------------------------------|
| status    | String    | The status of the transaction            |
| message   | String    | A message describing the transaction status |
| data      | Object    | An object containing details of the transaction |

## Security
To access this API endpoint, you will need to provide an API key in the header:

```javascript

  headers: { 
    "Content-Type": "application/json", 
    "authorization": "APIKEY"
  }
}
```

# Transaction Details

This endpoint retrieves detailed information for a specific transaction, identified by its unique transaction ID.

## Request Parameters
The request must be a POST request with the following parameters:

| Parameter | Data Type | Required | Description                    |
|-----------|-----------|----------|--------------------------------|
| id        | String    | Yes      | for getting the transaction details |

# Common Error Responses

Below are common HTTP error responses you may encounter when using the Transactions API:

| Status Code | Message                                          | Description                                                                 |
|-------------|--------------------------------------------------|-----------------------------------------------------------------------------|
| 404         | "Resource not found: GET /api/v1/public/transaction-lists" | The requested endpoint or resource does not exist. Double-check the URL or endpoint path. |
| 401         | "Authorization header missing"                   | The request is missing an Authorization header. Ensure you provide a valid token. |
| 401         | "You are unauthorized to perform this action."   | The credentials provided are not authorized to access this resource. Check your token or permissions. |

**Example Error Response (JSON)**:
```json
{
  "status_code": 401,
  "message": "Authorization header missing"
}
```

## Data Retention

> Transaction data is retained for the following periods:
> 
> - Complete transaction records: 2 years
> - Transaction summaries: 7 years
> - After the retention period, some sensitive data may be redacted while maintaining the transaction record
